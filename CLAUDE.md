## Project State

Use `docs/current-state.md` as the shared compact handoff.

- Update it only after meaningful project changes
- Keep only current truth in these sections:
  - Current systems
  - Known issues
  - Next 3 priorities
  - Files that matter most
- Replace stale bullets instead of appending logs or history
- Do not store changing app state in this file
- Do not store changing app state in `CLAUDE.md`
